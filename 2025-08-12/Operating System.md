## 📌 운영체제(OS) 종류와 특징
### 1. Windows
1. 사용자 친화적인 인터페이스

- 직관적이고 사용하기 쉬운 그래픽 인터페이스(GUI) 제공

- 일반 사용자부터 기업까지 널리 사용됨

2. 소프트웨어 호환성

- 다양한 상용·비즈니스 애플리케이션과 호환

3. 비즈니스 시장 지배

- 사무용, 기업용 환경에서 표준 OS로 자리잡음

### 2. macOS
1. 통합 생태계

- iPhone, iPad 등 Apple 기기와 연동 최적화

2. 디자인 중심

- 세련된 UI/UX, 멀티미디어 성능에 강점

3. 보안·안정성

- Unix 기반으로 안정적이고 보안성 높음

### 3. Linux
1. 오픈 소스

- 무료, 누구나 소스 코드 수정·배포 가능

2. 유연성과 사용자 정의

- 개발자, 서버 관리자, 일반 사용자 모두 사용 가능

3. 서버·임베디드·슈퍼컴퓨터 강점

- 높은 안정성, 확장성, 보안성

### 4. 모바일 운영체제
1. Android (AOS)

- Google 주도, 전 세계 시장 점유율 1위

- 오픈 소스 기반, 다양한 기기 제조사 지원

2. iOS

- Apple 전용, 폐쇄형 생태계

- 보안성, 안정성, 성능 최적화

### 5. Unix
- 다중 사용자·멀티태스킹

- 서버·워크스테이션 환경에서 설계

- 이식성·확장성·견고성

- 다양한 하드웨어에서 동작 가능

다른 OS의 기반

- macOS, Linux 등 Unix 철학 기반

## 📌 리눅스 기초 용어
### 1. 바이너리 (Binary)
- 실행 가능한 파일

- Windows의 .exe와 유사

- /bin, /usr/bin에 명령어 실행 파일 존재

### 2. 디렉토리 (Directory)
- 파일과 다른 디렉토리를 포함하는 구조(폴더 개념)

- 리눅스의 최상위 디렉토리는 / (루트 디렉토리)

### 3. 루트 (root)
- 관리자 계정 (슈퍼유저)

- 모든 파일·명령어 접근 가능

- 시스템 설정, 사용자 관리 가능

- root 권한 명령: sudo 명령어

### 4. 터미널 (Terminal)
- CLI(Command Line Interface) 환경에서 명령어 입력 창

- GUI 대신 텍스트 기반 명령으로 시스템 제어

### 5. 쉘 (Shell)
- 사용자의 명령어를 해석해 커널에 전달하는 프로그램

종류:

- bash (기본)

- zsh, fish 등

- 쉘 스크립트로 작업 자동화 가능

### 6. 절대경로 / 상대경로
1. 절대경로: /부터 시작하는 전체 경로
예) /home/user/project

2. 상대경로: 현재 디렉토리를 기준으로 한 경로
예) ../project, ./file.txt

## 📌 주요 명령어 정리
### ls (디렉토리 목록 보기)
| 옵션                     | 설명                  |
| ---------------------- | ------------------- |
| `ls -al`               | 모든 파일(숨김 포함) 자세히 표시 |
| `ls -t`                | 수정 시간 순 정렬 (최신 우선)  |
| `ls -at`               | 숨김 파일 포함, 수정 시간 순   |
| `ls -hl`               | 사람이 읽기 쉬운 크기 단위로 표시 |
| `ls /etc \| tail -n 5` | `/etc`의 하위 5개 항목 출력 |

### 디렉토리/파일 관리
| 명령어                      | 설명              |
| ------------------------ | --------------- |
| `mkdir fastcampus-linux` | 디렉토리 생성         |
| `mkdir -p a/b/c`         | 중간 폴더까지 한 번에 생성 |
| `rm -r folder`           | 폴더와 하위 내용 삭제    |
| `rm -rf folder`          | 강제 삭제 (경고 없음)   |

### 정보 확인
| 명령어                               | 설명                      |
| --------------------------------- | ----------------------- |
| `man pwd`                         | 명령어 매뉴얼 보기              |
| `whereis cat`                     | 명령어의 바이너리 위치 확인         |
| `which cat`                       | 실행 경로 확인                |
| `find /usr/bin -type f -name cat` | 특정 이름의 파일 검색            |
| `find /usr/local -name "python*"` | 이름이 python으로 시작하는 파일 검색 |

## 7. 파일 디스크립터(File Descriptor)

* 파일이나 소켓, 파이프 등 **열린 파일을 식별하는 정수 값**
* 파일을 사용하려면 먼저 `open()` 시스템 콜로 열어야 함
* **커널**은 `파일 테이블(File Table)`을 통해 열린 파일 정보를 관리
* 파일 테이블 항목에는 **메타데이터(파일 크기, 접근 모드, 위치 등)** 포함

# 📌 시스템 콜 & 프로세스 / 스레드 개념 정리

## 1. 시스템 콜(System Call) 추적 도구: `strace`

`strace`는 리눅스에서 프로그램이 호출하는 **시스템 콜**을 추적하는 도구입니다.
프로그램이 커널과 어떤 방식으로 상호작용하는지 확인할 때 사용합니다.

### 기본 사용법


strace COMMAND


예시:
strace ls


### 주요 옵션

| 옵션              | 설명                    |
| --------------- | --------------------- |
| `-o <파일명>`      | 추적 결과를 파일로 저장         |
| `-t`            | 타임스탬프 출력              |
| `-tt`           | 밀리초 단위 타임스탬프 출력       |
| `-T`            | 각 시스템 콜 소요 시간 출력      |
| `-c`            | 시스템 콜별 호출 횟수/시간 통계 출력 |
| `-e trace=<콜명>` | 특정 시스템 콜만 필터링         |

예시:


# 결과를 sample_out.txt에 저장
strace -o sample_out.txt ls

# read 시스템 콜만 추적
strace -e trace=read ls




## 2. 주요 시스템 콜

| 시스템 콜                | 설명                         |
| -------------------- | -------------------------- |
| `open`               | 파일 열기 (파일 디스크립터 반환)        |
| `close`              | 파일 닫기 (자원 해제)              |
| `read`               | 파일 디스크립터에서 데이터 읽기          |
| `write`              | 파일 디스크립터에 데이터 쓰기           |
| `fork`               | 새로운 프로세스(자식) 생성            |
| `exec*`              | 프로세스 메모리 공간을 새로운 프로그램으로 교체 |
| `getpid` / `getppid` | 현재/부모 프로세스 PID 반환          |
| `syslog`             | 시스템 로그 메시지 남기기             |
| `exit`               | 프로세스 종료                    |

💡 **파일 디스크립터(File Descriptor)**: 운영체제가 파일, 소켓 등을 구분하기 위해 부여하는 정수 ID.

## 3. 프로세스(Process)

**정의:** 실행 중인 프로그램의 인스턴스

### 프로세스 제어 블록(PCB, Process Control Block)

* PID / PPID (프로세스 ID / 부모 프로세스 ID)
* 레지스터 상태
* 스케줄링 정보
* 메모리 정보
* 열린 파일 정보
* 입출력 장치 상태

### 프로세스 메모리 구조

| 영역            | 설명                                    |
| ------------- | ------------------------------------- |
| **코드(Code)**  | 실행 가능한 기계어 명령 (Read-Only)             |
| **데이터(Data)** | 전역 변수, 정적 변수 (초기값 있는 데이터)             |
| **BSS**       | 초기값 없는 전역/정적 변수                       |
| **힙(Heap)**   | 런타임 시 동적 메모리 할당 (낮은 주소 → 높은 주소)       |
| **스택(Stack)** | 함수 호출 시 지역변수, 매개변수 저장 (높은 주소 → 낮은 주소) |

---

## 4. 프로세스 상태

| 상태             | 설명                              |
| -------------- | ------------------------------- |
| 생성(Create)     | 프로세스 생성 중                       |
| 준비(Ready)      | CPU 할당 대기 중                     |
| 실행(Running)    | CPU에서 실행 중                      |
| 대기(Waiting)    | I/O 등 이벤트 대기 중                  |
| 종료(Terminated) | 실행 완료 후 종료됨                     |
| 좀비(Zombie)     | 종료되었지만 부모가 `wait()`로 수거하지 않은 상태 |

리눅스에서 프로세스 상태 코드:

* **R**: Running
* **S**: Sleeping
* **W**: Waiting
* **T**: Stopped
* **Z**: Zombie

---

## 5. 문맥 교환(Context Switch)

CPU가 다른 프로세스로 작업을 전환하기 위해 **PCB**를 저장하고 불러오는 과정.
여러 프로세스가 CPU를 번갈아 사용하는 핵심 메커니즘.

---

## 6. 스레드(Thread)

**정의:** 프로세스 내부에서 실행되는 **작업 흐름의 단위**

### 특징

* 같은 프로세스의 스레드는 메모리(코드, 데이터, 힙)를 공유
* 스레드마다 개별 스택, 레지스터, 프로그램 카운터 보유

---

## 7. 멀티프로세스 vs 멀티스레드

| 구분      | 멀티프로세스                    | 멀티스레드                 |
| ------- | ------------------------- | --------------------- |
| 자원 공유   | 기본적으로 공유 안 함              | 프로세스 자원 공유            |
| 메모리 사용량 | 큼                         | 적음                    |
| 안정성     | 한 프로세스 오류 → 다른 프로세스 영향 없음 | 한 스레드 오류 → 전체 프로세스 영향 |
| 속도      | 느림 (문맥 교환 비용 큼)           | 빠름 (공유 메모리 접근)        |

---

## 8. 프로세스 간 통신(IPC, Inter-Process Communication)

* **공유 메모리(Shared Memory)**: 메모리 영역 공유
* **파이프(Pipe)**: 한 방향 통신 채널
* **소켓(Socket)**: 네트워크 기반 양방향 통신
